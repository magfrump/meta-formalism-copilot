# User Research Action Items

Tracking document for issues surfaced during user research sessions.

## High Priority (UX blockers)

- [x] **Hide semiformal and Lean tabs by default** — Users shouldn't see empty output tabs before formalization runs. *(u0 #4)*
- [ ] **Fix decompose buttons** — Buttons didn't respond during session. Needs debugging. *(u0 #7)*
- [ ] **Clean up input tab layout** — Input section is too busy; context box text is hard to read. Needs layout/spacing work. *(u0 #3)*
- [ ] **Remove or defer refinement chips on context box** — Chips appearing immediately after typing feels like premature output. Show them only after formalization, or move them elsewhere. *(u0 #5)*
- [x] **Don't auto-select "deductive (lean)" chip** — Pre-selection obscures what the user is choosing. Default to no selection or make the selection step more explicit. *(u0 #2)*

## Medium Priority (feature gaps)

- [ ] **Add formalism descriptions/tooltips** — Each formalism type needs an explanation of what it is and when it's appropriate. *(u0 #9)*
- [ ] **Make new formalism results exportable** — Users need to get results out of the app. *(u0 #12)*
- [ ] **Add progress indicator for formalization** — Show estimated time remaining during long waits. Consider persisting analytics data (with `.gitignore`) to improve estimates. *(u0 #8)*
- [ ] **Session management** — Add "new session" button that resets all state, plus a dropdown to return to previous sessions. *(u0 #14)*
- [ ] **Design unique interfaces per formalism** — Each formalism type should have a tailored display, not a generic output panel. *(u0 #13)*

## Low Priority (onboarding / future)

- [ ] **Onboarding guidance** — Prompt users to bring a document or idea. Could be a landing state or tooltip on the input panel. *(u0 #1)*
- [ ] **Wiki-link formalism terms** — Users want to explore referenced concepts. Could link technical terms to Wikipedia or similar. *(u0 #10)*

## Positive Signals

- User called the results an "excellent backbone for a book" *(u0 #11)* — core value proposition is landing even with UX friction.
- User tried a "statistical model" for "do myths make us human" *(u0 #6)* — creative use of formalisms beyond expected patterns.
